# SignSpeak 🤟🔊

**Real-time Indian Sign Language (ISL) → text → speech translator.**

A webcam watches a person sign, the signs appear as text on screen, and the computer speaks them out loud — giving a voice to ISL users when talking to people who don't know sign language.

Built with Python, OpenCV, MediaPipe and scikit-learn. Runs in real time (~25 fps) on a normal laptop CPU — no GPU needed.

## Why

India has an estimated 5–18 million deaf people and only ~250–300 certified ISL interpreters — and almost all sign-language tech (datasets, tutorials, tools) targets *American* Sign Language. ISL is structurally different (two-handed fingerspelling, SOV grammar) and underserved. SignSpeak is a step toward free, accessible, **two-way** ISL communication — and the dataset used here is self-recorded for ISL specifically.

## How it works

```
webcam frame → MediaPipe Hands (21 3D landmarks per hand)
            → normalized landmark features (wrist-relative, scale-invariant)
            → classifier predicts the sign
            → stability filter (a sign must hold for N frames to register)
            → sentence buffer → text-to-speech
```

## Project structure

```
src/signspeak/
  capture.py     # webcam I/O boundary (the only module that opens a camera)
  landmarks.py   # MediaPipe boundary (the only module that imports mediapipe)
  features.py    # pure: landmarks → normalized 128-dim feature vectors
  sentence.py    # pure: debouncing state machine + sentence buffer
  predict.py     # pure: features → (label, confidence)
  training.py    # pure: dataframe → trained model + metrics
  config.py      # configs/*.yaml → typed access
  tts.py         # threaded text-to-speech
  cli/           # entry points: track / collect / train / live
configs/         # all tunables (thresholds, seeds, paths)
tests/           # headless test suite — runs with no camera
data/            # recorded landmark datasets (CSV; landmarks only, never video)
models/          # trained models (regenerated by signspeak-train)
```

## Setup

```bash
python -m venv venv && source venv/bin/activate   # Windows: venv\Scripts\activate
pip install -e ".[dev]"
```

## Usage

```bash
signspeak-track                     # 1. verify webcam + hand tracking works
signspeak-collect --label HELLO --signer S01   # 2. record samples (press 's')
signspeak-train                     # 3. train + per-sign accuracy report
signspeak-evaluate                  # 4. full honest report -> reports/eval_report.md
signspeak-live                      # 5. live translation: signs → text → speech
signspeak-record-seq --label HELLO  # (Phase 3) record dynamic-sign sequences
```

In `signspeak-live`, a recognized sign appears as a **candidate** first — ENTER
confirms it into the sentence, `x` rejects it, SPACE speaks the sentence. The
signer stays in control of what gets spoken.

Optional extras:

```bash
pip install -e ".[online]"   # edge-tts neural Indian English/Hindi voices
pip install -e ".[stt]"      # Vosk speech-to-text for two-way mode
pip install -e ".[dynamic]"  # TensorFlow for the dynamic word-sign tier
```

## Performance

Measured on a laptop CPU (x86_64, 1000 runs, `benchmarks/latency.py`):

| Stage | p50 | p95 |
|---|---|---|
| feature vectorization | 0.06 ms | 0.09 ms |
| classifier inference | 8.5 ms | 10.1 ms |

MediaPipe landmark extraction adds ~16–33 ms/frame (its documented CPU cost),
keeping the full capture→prediction path within the 70 ms budget (NFR-2).

## Dataset note

All training data is self-recorded landmark data (no images/video are stored — only the 21-point hand geometry per sample), collected with consenting volunteers. This keeps the dataset small, privacy-friendly, and ISL-specific.
